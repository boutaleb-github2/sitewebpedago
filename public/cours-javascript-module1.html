Aide-mémoire JavaScript – Module 1

Objectif : se rappeler l’essentiel sur ce qu’est JavaScript, comment il s’exécute, les variables et les types de base.

1. Rôle de JavaScript

Qu’est-ce que JavaScript ?

  • Langage de programmation du web.
  • S’exécute :
    • dans le navigateur → rendre les pages interactives, gérer les formulaires, boutons, menus…
    • dans Node.js → côté serveur (back-end).
  • Permet de :
    • modifier le contenu d’une page sans la recharger,
    • communiquer avec des API (envoyer/recevoir des données),
    • créer des applications complètes (React, etc.).
2. Exécution

Comment JavaScript s’exécute ?

  • Le navigateur ou Node.js contient un moteur JS (ex. V8).
  • Le moteur lit ton code ligne par ligne.
  • Il utilise une pile d’exécution (call stack) :
    • Quand une fonction est appelée, elle est ajoutée en haut de la pile.
    • Une fois exécutée, elle est retirée de la pile.
console.log("Hello");
// 1. 'console.log' est placé dans la pile
// 2. le message est affiché
// 3. 'console.log' est retiré de la pile
3. Variables

Variables : let, const, var

Pourquoi des variables ?

Pour stocker des informations : nom, âge, configuration, résultats de calculs, etc.

const (valeur qui ne change pas)

  • Pour les valeurs qui ne doivent pas être réassignées.
  • Très utilisé dans le code moderne.
const apiBaseUrl = "https://api.exemple.com";
// apiBaseUrl = "https://autre-api.com"; // ❌ erreur

let (valeur qui peut changer)

  • Pour les valeurs qui évoluent avec le temps.
let count = 0;
count = count + 1;

var (ancien, à éviter)

  • Gère mal les blocs (if, for…), provoque des comportements surprenants.
  • À connaître pour lire du vieux code, mais ne plus utiliser dans le nouveau.
4. Types de base

Types de données essentiels

1) number – nombres

let age = 30;
let price = 19.99;

2) string – texte

  • Très utilisé pour les messages, labels, champs de formulaires…
let name = "Habib";
let message = "Bonjour " + name;
let message2 = `Bonjour ${name}`; // template string

3) boolean – vrai / faux

  • Base de la logique : conditions, affichage, permissions…
let isLoggedIn = true;
let isAdmin = false;

4) undefined vs null

  • undefined : aucune valeur définie (par défaut).
  • null : on met volontairement “rien”.
let x;        // x est undefined
let y = null; // y est "vide" volontairement

5) Aperçu des object

Très important en programmation web (représenter un utilisateur, un produit, un message, etc.).

const user = {
  name: "Habib",
  age: 50,
  isAdmin: false
};
5. Erreurs

Erreurs courantes

  • JavaScript affiche des messages d’erreur dans la console du navigateur.
  • Lire le message aide beaucoup pour comprendre ce qui ne va pas.
console.log(nom);
// ReferenceError: nom is not defined
// → la variable 'nom' n'existe pas
6. En résumé

Résumé rapide du Module 1

  • JS = langage pour rendre le web interactif.
  • Le code s’exécute dans un moteur JS (navigateur ou Node.js) avec une pile d’exécution.
  • const : valeur fixe, let : valeur qui change, var : ancien, à éviter.
  • Types essentiels :
    • number, string, boolean, undefined, null, object.
  • Lire les messages d’erreur dans la console est une compétence importante.
